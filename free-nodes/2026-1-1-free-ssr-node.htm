详细图文教程：添加订阅的7个步骤

下面我们以iOS系统为例，详细介绍Quantumult X添加订阅的全过程：

在手机主屏找到紫色图标的Quantumult X应用。首次打开时，系统可能会请求VPN配置权限，务必点击"允许"。

步骤2：进入配置界面

点击底部导航栏的"设置"齿轮图标 → 选择"配置"选项。这里是Quantumult X的核心控制中心。

步骤3：访问节点管理

在配置界面右上角找到"节点"按钮。这里会显示当前所有可用的代理节点和订阅列表。

步骤4：添加新订阅

点击右上角的"+"号 → 选择"添加订阅"。此时会弹出URL输入框。

专业技巧：长按输入框可以选择"粘贴"，如果您已经复制了订阅链接，系统会自动识别。

步骤5：输入订阅链接

将事先准备好的订阅链接完整粘贴到输入框中。典型的订阅链接格式类似：
https://example.com/subscribe?token=your_unique_key

重要提示：
- 确保链接完整无遗漏
- 检查是否有特殊字符被自动转换
- 付费服务商的链接通常包含个人识别码，不要随意分享

步骤6：确认并保存

点击"确定"后，Quantumult X会立即尝试连接订阅服务器。如果链接有效，您会看到"订阅添加成功"的提示。

步骤7：首次同步节点

返回节点列表，下拉页面触发手动更新，或点击右上角的刷新图标。稍等片刻，节点列表就会自动填充。

性能优化建议：在Wi-Fi环境下进行首次同步，特别是包含大量节点的订阅可能需要较长时间。

FAQ

SSR 的 UDP 转发功能需要注意什么？
SSR 的 UDP 转发需要服务端和客户端同时开启，同时确保服务器防火墙允许 UDP 通信。UDP 流量不可靠，可能在高丢包网络中导致连接中断，需要测试稳定性。
Clash 的 Rule Provider 自动更新规则如何操作？
Rule Provider 可以通过远程 URL 自动下载规则和节点，定期更新无需手动操作。结合策略组使用，可保持节点和分流规则最新，提高访问速度和网络可用性。
Netch 的 Socks 模式有什么作用？
Netch 的 Socks 模式允许用户将代理仅应用于指定进程，而非全局网络。此模式非常适合单个游戏或程序代理，避免影响其他应用的连接方式与速度。
Netch 的全局代理与规则代理有何不同？
全局代理将所有流量都通过节点转发，而规则代理只针对指定 IP、端口或应用。全局代理适合游戏和全局科学上网，规则代理适合优化带宽和分流特定流量。
Trojan 的连接失败常见原因有哪些？
Trojan 连接失败可能由于伪装域名错误、TLS 证书过期、防火墙阻挡端口或服务器不可达。通过查看客户端日志可以定位具体原因，并对症下药解决问题。
Shadowrocket 的日志界面怎么看节点状态？
Shadowrocket 的日志界面实时显示连接记录与节点响应延迟。绿色代表成功连接，红色代表失败，通过查看错误码可判断是 DNS 问题、TLS 握手失败还是代理超时。
Sing-box 如何保证规则订阅的实时更新？
Sing-box 支持远程规则订阅 URL，可设置定期自动更新频率，如每小时或每天。自动更新保证节点和规则保持最新，减少手动维护，提高访问速度和分流效率。
WinXray 的多节点同时使用会影响性能吗？
同时使用多个节点会占用更多系统资源和带宽，可能对性能有一定影响。通过策略组和分流规则合理安排流量，可以降低资源占用，同时保持网络稳定和速度。
Hysteria2 多路复用对即时通讯的优势是什么？
多路复用可让多个短连接共享同一通道，减少握手次数和加密开销，提高吞吐量和连接稳定性。适合即时通讯应用，保证消息快速传输和稳定接收。
Nekoray 节点订阅失败如何解决？
确认订阅 URL 是否有效并检查网络通畅。可手动刷新或重新添加订阅链接，保证节点列表最新。结合策略组使用，可确保稳定科学上网体验。
